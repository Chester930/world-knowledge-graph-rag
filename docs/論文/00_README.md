本資料夾存放正式論文的段落文件草稿。目前狀態：**尚未正式動筆，這是寫作前的骨架準備**。

## 與既有素材的關係

- 既有的整理資料在 `智慧知識庫`（v1）的 `docs/報告/`：
  - `01_專案總覽_是什麼為什麼怎麼做.md` — 系統設計論證鏈，可作為第三章素材
  - `02_參考文獻獨立查核報告.md` — 38 條引用查核，可作為第二章文獻回顧與參考文獻的底稿
  - `03_參考來源連結清單.md` — 引用連結，寫作時核對原文用
  - `04_對標NotebookLM_不足分析與完全超越方案.md` — **產品競爭分析，非學術論文素材**，僅部分技術構想（向量引導圖剪枝等）可轉化為研究設計
  - `docs/THEORETICAL_ARCHITECTURE.md` — 完整理論架構與本體設計（T-Box/A-Box）
- `World Knowledge Graph RAG`（本專案）是論文的**實作載體**。**專案定位（2026-07-10 確認）**：本專案不是 v1 的延伸開發，而是在 v1 已驗證核心概念可行性與價值的前提下，重新、更全面地準備一套完整的專案與論文，架構已重整為更嚴謹的分層（見 `../ARCHITECTURE.md`）
- 本專案自己的 `docs/報告/`（v2 新建）：
  - `產品競品研究/` — 知識圖譜/GraphRAG/個人知識管理領域的競品調查（NotebookLM、Neo4j Builder、Zep、Glean、RAGFlow、Stardog、WhyHow.ai、Obsidian 生態，每個競品各一份獨立文件 + `00_總覽.md` 統整），**產品規劃材料，非學術論文素材**，供 1.1.4 節結論引用時參考
  - `技術導入評估.md` — 針對前述競品調查中「值得借鏡的技術點」逐項做正式評估（是否採用、參考開源實作、學術文獻支撐），**產品/工程規劃材料，非學術論文素材**
  - `技術參考地圖.md` — 逐層記錄「本專案哪個功能主要參考哪個專案、補強哪些重點、在哪個分支導入」的持續累積追蹤表，是 02 文獻探討與 03 系統設計方法論的素材庫，**目前是空白格式骨架，隨實作進度逐步填入**
