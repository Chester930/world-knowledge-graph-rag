# 技術參考地圖

> 目的：逐層記錄「本專案哪個功能/元件，主要參考了哪個外部專案，針對哪些重點做了補強，實際在哪個分支/PR 導入」。這是**持續累積**的追蹤表，不需要一次填滿——每次真正把某個借鏡技術落地實作時，才登記一列。
>
> 與既有文件的分工：
> - `產品競品研究/` — 競品調查，回答「市面上有哪些做法」
> - `技術導入評估.md` — 逐項技術的「要不要採用」評估，回答「值不值得做」
> - **本文件** — 只記錄**已經或正在實作**的項目，回答「實際做了什麼、參考誰、在哪一版」
>
> 用途：這份表格是 02 文獻探討（哪些設計選擇有對應文獻/前例）與 03 系統設計與方法論（哪些架構決策的參考依據）的**素材庫**，寫作時直接從這裡撈資料，不用重新回憶當初為什麼這樣設計。
>
> 專案定位提醒：本專案（v2）**不是**智慧知識庫（v1）的延伸開發，而是在 v1 已驗證核心概念可行性與價值的前提下，重新、更全面地準備一套完整的專案與論文。因此下表的「主要參考專案」欄，v1 也可以是一個條目（視為「已驗證但需要重新設計」的參考來源，不是「直接沿用」的來源），需在補強重點欄註明是重新設計還是延用邏輯。
